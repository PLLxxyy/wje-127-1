## 根因分析（本次修复的问题）

### 问题原因
管理员详情页的"开始处理"和"标记已修好"按钮使用了以下错误逻辑：
```javascript
onClick={() => {
  setNewStatus('processing');      // React 异步 state 更新
  setTimeout(() => handleUpdate(), 0);  // 立即执行，但 state 还没更新
}}
```

在 `handleUpdate` 中检查：
```javascript
if (newStatus !== repair!.status) payload.status = newStatus;
```

由于 `setNewStatus` 是 React 的异步 state 更新，`setTimeout(fn, 0)` 执行时 `newStatus` 仍是旧值，导致 `newStatus !== repair.status` 条件为 false，状态没有被加入提交 payload。

### 修复方案
新增独立的 `handleQuickStatusChange` 函数，直接传入目标状态，不依赖异步 state 更新：
```javascript
const handleQuickStatusChange = async (targetStatus: string) => {
  await api.put(`/admin/repairs/${id}`, { status: targetStatus });
  await fetchRepair();  // 提交后立即刷新，获取最新日志
};
```

按钮点击改为：
```javascript
onClick={() => {
  setNewStatus('processing');      // 仅用于同步下拉框显示
  handleQuickStatusChange('processing');  // 直接提交，不依赖 state
}}
```
